# Node.jsが認識されない場合の最終的な解決方法

## 現在の状況

- Node.jsはインストールされている（`C:\Program Files\nodejs\`フォルダが存在）
- PATH環境変数にも追加されている
- しかし、ターミナルで `node` コマンドが認識されない

## 解決方法（順番に試してください）

### 方法1: PCを再起動（最も確実）

環境変数の変更を完全に反映するには、PCの再起動が必要な場合があります。

1. **PCを再起動**
2. 再起動後、新しいターミナル（PowerShell）を開く
3. 以下を実行：
   ```powershell
   node --version
   npm --version
   ```

### 方法2: 完全なパスで直接実行して確認

PCを再起動する前に、Node.jsが正しくインストールされているか確認：

```powershell
& "C:\Program Files\nodejs\node.exe" --version
& "C:\Program Files\nodejs\npm.cmd" --version
```

これでバージョンが表示されれば、Node.jsは正しくインストールされています。問題はPATHの反映です。

### 方法3: 環境変数の再確認

1. Windowsキー + R → `sysdm.cpl` → Enter
2. 「詳細設定」→「環境変数」
3. **「ユーザー環境変数」**と**「システム環境変数」**の両方で「Path」を確認
4. どちらかに `C:\Program Files\nodejs` が含まれているか確認
5. 含まれていない場合は追加
6. **PCを再起動**

### 方法4: 手動でPATHを更新（PowerShellで）

管理者権限でPowerShellを開いて：

```powershell
[Environment]::SetEnvironmentVariable("Path", $env:Path + ";C:\Program Files\nodejs", [EnvironmentVariableTarget]::Machine)
```

その後、**PCを再起動**

## 推奨される手順

1. **まず方法2で、Node.jsが正しくインストールされているか確認**
2. **方法1でPCを再起動**
3. それでもダメな場合は、方法3で環境変数を再確認

## 確認方法

正しく設定されていれば、以下のコマンドでバージョンが表示されます：

```powershell
node --version
# 例: v20.11.0

npm --version
# 例: 10.2.4
```
