# Node.jsとnpmのインストール手順

このプロジェクトを実行するには、**Node.js**と**npm**が必要です。

## エラーの原因

`npm: 用語 'npm'は、コマンドレット、関数、スクリプトファイル、または操作可能なプログラムの名前として認識されません。`

このエラーは、Node.jsとnpmがインストールされていないか、PATH環境変数に追加されていないことを意味します。

## 解決方法: Node.jsをインストールする

### ステップ1: Node.jsの公式サイトにアクセス

1. ブラウザで以下のURLを開きます：
   ```
   https://nodejs.org/ja/
   ```

2. または、英語版のサイト：
   ```
   https://nodejs.org/
   ```

### ステップ2: インストーラーをダウンロード

1. サイトにアクセスすると、**2つのバージョン**が表示されます：
   - **LTS版**（推奨）: 安定版（例: v20.x.x）
   - **最新版**: 最新機能を含む版

2. **「LTS版」の「ダウンロード」ボタン**をクリック
   - 自動的にWindows用のインストーラー（`.msi`ファイル）がダウンロードされます
   - ファイル名は `node-v20.x.x-x64.msi` のような形式です

### ステップ3: インストーラーを実行

1. ダウンロードした`.msi`ファイルをダブルクリック
2. セキュリティ警告が出た場合は「実行」をクリック
3. インストールウィザードが開きます：
   - 「Next」をクリック
   - 利用規約に同意して「Next」をクリック
   - インストール先を選択（デフォルトのままでOK）→「Next」
   - **重要**: 「Add to PATH」のチェックボックスが**ONになっていることを確認**（通常はデフォルトでON）
   - 「Next」をクリック
   - 「Install」をクリック
   - 管理者権限の確認が出たら「はい」をクリック
   - インストールが完了したら「Finish」をクリック

### ステップ4: インストールの確認

1. **現在開いているPowerShellやターミナルをすべて閉じる**
   - これは重要です！新しいPATH設定を読み込むために必要です

2. **新しいPowerShellまたはターミナルを開く**

3. 以下のコマンドを実行して、インストールを確認：

```powershell
node --version
```

正常にインストールされていれば、バージョン番号が表示されます（例: `v20.11.0`）

4. npmも確認：

```powershell
npm --version
```

バージョン番号が表示されればOKです（例: `10.2.4`）

### ステップ5: プロジェクトのセットアップを再開

インストールが確認できたら、プロジェクトフォルダに移動して依存パッケージをインストール：

```powershell
cd project-bolt-sb1-pcvnpemp/project
npm install
```

## トラブルシューティング

### インストール後も「npm が認識されない」エラーが出る場合

1. **すべてのターミナル/PowerShellを閉じる**
2. **PCを再起動**（最も確実な方法）
3. 新しいターミナルを開いて再度試す

### それでも解決しない場合

1. **環境変数の確認**：
   - Windowsキー + R を押す
   - `sysdm.cpl` と入力してEnter
   - 「詳細設定」タブ → 「環境変数」をクリック
   - 「システム環境変数」の「Path」を選択 → 「編集」
   - 以下のパスが含まれているか確認：
     - `C:\Program Files\nodejs\`
   - 含まれていない場合は追加

2. **Node.jsの再インストール**：
   - コントロールパネルからNode.jsをアンインストール
   - 上記の手順で再インストール

## 代替方法: Chocolateyを使用（上級者向け）

Chocolateyがインストールされている場合：

```powershell
choco install nodejs
```
