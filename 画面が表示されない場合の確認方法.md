# 画面が表示されない場合の確認方法

## 確認手順

### ステップ1: ブラウザのコンソールを確認

1. **ブラウザでF12キーを押す**（開発者ツールを開く）
2. **「Console」タブを開く**
3. **エラーメッセージを確認**

よくあるエラー：
- `VITE_DIFY_API_KEY が設定されていません` → 環境変数が読み込まれていない
- `Failed to fetch` → API接続エラー
- `Cannot read property 'xxx' of undefined` → コードエラー

### ステップ2: 開発サーバーを再起動

環境変数（.envファイル）を変更した場合、**開発サーバーを再起動する必要があります**。

1. **ターミナルで `Ctrl + C` を押して開発サーバーを停止**
2. **再度起動：**
   ```powershell
   npm run dev
   ```

### ステップ3: .envファイルの確認

`.env`ファイルが正しく設定されているか確認：

1. **プロジェクトフォルダで`.env`ファイルを開く**
2. **以下の形式になっているか確認：**
   ```
   VITE_DIFY_API_KEY=app-xxxxxxxxxxxxx
   VITE_DIFY_API_ID=xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
   VITE_DIFY_API_BASE_URL=https://api.dify.ai/v1
   ```

**注意点：**
- `=`の前後にスペースを入れない
- 値の前後に引用符（`"`や`'`）を付けない
- 空行やコメント行（`#`で始まる行）は問題ない

### ステップ4: ネットワークタブを確認

1. **開発者ツール（F12）の「Network」タブを開く**
2. **ページをリロード（F5）**
3. **エラーが出ているリクエストを確認**

## よくある原因と解決方法

### 原因1: 環境変数が読み込まれていない

**症状：** コンソールに `VITE_DIFY_API_KEY が設定されていません` と表示される

**解決方法：**
1. `.env`ファイルがプロジェクトルート（`project`フォルダ）にあるか確認
2. ファイル名が`.env`であることを確認（`.env.txt`ではない）
3. 開発サーバーを再起動

### 原因2: 環境変数の値が空

**症状：** コンソールにエラーが表示される

**解決方法：**
1. `.env`ファイルを開いて、値が正しく設定されているか確認
2. `=`の後に値が入っているか確認

### 原因3: JavaScriptエラー

**症状：** コンソールにJavaScriptエラーが表示される

**解決方法：**
1. エラーメッセージを確認
2. エラーの内容を報告してください

## デバッグ用の一時的な修正

環境変数が正しく読み込まれているか確認するため、一時的に以下を追加：

`src/App.tsx`に以下を追加：
```tsx
console.log('API Key:', import.meta.env.VITE_DIFY_API_KEY);
console.log('App ID:', import.meta.env.VITE_DIFY_APP_ID);
```

ブラウザのコンソールで値が表示されれば、環境変数は読み込まれています。
